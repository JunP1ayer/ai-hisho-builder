# 構築指示書 第3部「中身」— 会社を理解させる

> **この指示書の上に書かれている「あなたの決定」（引き継ぎ済み資料一覧・自社理解度スコア）を読んでから、以下の手順を実行すること。**
> 第1部で体、第2部で能力ができた。第3部では `context/` の中身を充実させ、AI秘書に会社を理解させる。ここが薄いと、どれだけスキルを作っても的外れな仕事しかできない。

## 1. context/ の考え方：ストックとフローを分ける

- **ストック**＝今の正しい状態。会社概要・料金表・顧客リストなど、変わるたびに**上書き更新**する情報。`context/`直下（company/ customers/ products/ faq/ 等）に置く
- **フロー**＝議事録・作業ログなど、時系列で**追記**していく情報。古い記録を消さず積み上げる。`work/`側、または`context/`内でも`log/`のような専用フォルダに分けて置く

この2つを同じファイルに混ぜない。ストックのファイルに日々のメモを書き足すと、どこが「今の正しい情報」か分からなくなる。

## 2. 資料を1つずつcontext/へ入れる

「あなたの決定」に書かれた引き継ぎ済み資料（会社概要・沿革／商品・サービス資料／料金表・見積ひな形／顧客リスト／よくある質問と回答／社内用語集／過去の提案書サンプル 等）を、次の手順で1件ずつ処理する。

1. 資料の所在を担当者に確認する（既存ファイルがある／口頭で説明してもらう しかない、のどちらか）
2. **ファイルがある場合**：ファイルを置いてもらう場所（`work/`直下でよい）を伝え、配置後に読み込む
3. **ファイルがない場合**：会話で聞き書きする。担当者に質問し、回答をそのまま構成し直す
4. 読み込んだ内容・聞き書きした内容を整形し、`context/`内の適切なサブフォルダに保存する（例：料金表→`context/products/`、顧客リスト→`context/customers/`）
5. 保存したら「〈資料名〉を`context/◯◯/`に保存しました」と1件ずつ短く報告する
6. 次の資料に進む。全件終わったら4節のスコアを算出する

聞き書きで整形するときは、担当者の言葉を勝手に脚色せず、事実だけを構成し直す。憶測で数字や固有名詞を補わない。

### 進め方の例（「料金表・見積ひな形」の場合）

1. 「料金表の元ファイルはありますか？」と担当者に確認する
2. あれば`work/`に置いてもらい、読み込んで`context/products/pricing.md`に整形保存する。なければ「主なプラン名と価格帯を教えてください」等、聞き書きで最低限の項目を埋める
3. 保存後、「料金表・見積ひな形を`context/products/`に保存しました」と報告する
4. 内容に矛盾・古そうな情報（例：終了済みのプランが残っている等）があれば、その場で指摘し担当者に確認する

## 3. すべて終えたら要約をまとめる

引き継ぎ資料を一通り`context/`へ保存し終えたら、要点（会社の強み・商品・料金の考え方・顧客特性・よくある質問）を`context/summary.md`に1ファイルへ要約する。これはAI秘書自身が次回セッション以降に最初に目を通す「早見表」になる。個々の資料ファイルを削除する必要はない（summary.mdはあくまで要約、原本は残す）。

## 4. 自社理解度スコアを出す

引き継ぎ済み資料の充足率から0〜100点を算出する。

```
スコア = round(引き継ぎ済み資料の件数 ÷ 資料カテゴリ総数 × 100)
```

算出後、次のフォーマットで報告する。

```markdown
## 自社理解度スコア：<点数>点／100点

引き継ぎ済み：<件数>／<総数>
未引き継ぎ：<まだ無いカテゴリを列挙>

次に足すと効く資料：「<未引き継ぎカテゴリの中から1つ>」
```

「次に足すと効く資料」は、未引き継ぎの中から**担当業務（配属先）で使う頻度が高そうなもの**を1つ選ぶ（全部を一度に要求しない。1つに絞ることで担当者が動きやすくなる）。全カテゴリが揃っていれば「資料は揃っています。今後は内容の更新があれば都度反映してください」と伝える。

## 5. 機密の扱い（再確認）

- 顧客名・金額・個人情報などの機密情報は、`context/`に**保存してよい**（AI秘書が正しく業務を行うために必要）
- ただし、それらを**外部に送る文書・メッセージ（メール・提案書・SNS投稿等）には出さない**。これは第1部のCLAUDE.md絶対ルール#3と同じ内容であり、`context/`が充実するほど誤って混入させないよう改めて意識する
- 迷ったら「この情報は社外に出してよいか」を担当者に確認してから使う
